# 客户跟进提醒系统

面向外贸/出口业务的轻量级本地 CRM 系统，帮助 B2B 外贸业务员管理客户信息、记录跟进历史、设置待办提醒、跨设备同步数据，并支持 AI 辅助背调分析。

---

## 快速开始

系统启动时会自动创建虚拟环境并安装依赖，首次启动稍慢，后续即开即用。

### 方式一：双击 start.bat（推荐）

```
双击 → start.bat
```

会自动完成：创建虚拟环境 → 安装依赖 → 启动服务 → 打开浏览器。

### 方式二：双击 desktop.py（桌面应用模式）

```bash
python desktop.py
```

启动后以无框浏览器窗口打开，更像一个独立桌面软件。

### 手动启动

```bash
python app.py
```

然后手动访问 http://localhost:8080

### 停止服务

在启动窗口中按 `Ctrl + C`，系统会自动保存数据后退出。

---

## 第一次使用

1. 打开系统后，点击顶部「📊 上传 Excel」按钮
2. 选择你的客户跟进表格（.xlsx 或 .xls 格式）
3. 系统自动导入所有客户数据并生成跟进提醒

**表格推荐包含的列**：客户名称、国家、客户等级、客户类型、状态等。系统支持智能列名匹配，大部分常见列名都能自动识别。

首次启动时系统还会自动插入 5 个演示客户，方便你快速体验功能。

## 功能详解

### 客户管理

- **完整信息**：客户名称、公司、国家（自动转中文）、等级（A/B/C/C+/D）、类型（中间商/终端）、行业、网站、简介、公司规模、年营收等 20+ 字段
- **状态流转**：未建联 → 已建联 → 跟进中 → 成交 / 流失
- **批量操作**：批量修改状态、批量修改等级、批量删除
- **搜索筛选**：按名称、公司、国家、行业搜索

### 提醒系统

- 手动设置跟进日期，自动生成待办提醒
- 新客户自动生成 15 天 / 30 天 / 60 天开发信提醒
- 完成提醒时自动记录跟进历史并设置下次跟进日期
- 支持批量完成提醒
- 每日 9:00 Windows 系统 Toast 通知提醒今日待办（需保持服务运行）

### 跟进历史

- 每次跟进自动记录时间、内容、结果和下一步计划
- 支持编辑和删除历史记录
- 全局跟进时间线（最新 50 条）

### 联系人管理

- 每个客户可添加多个联系人
- 支持标记主要联系人
- 记录姓名、职位、邮箱、电话、LinkedIn

### 开发信管理

- 记录发送给客户的开发信，跟踪主题和内容
- 回复状态跟踪：待回复 / 已回复 / 退信 / 无回复

### AI 背调分析

> 需要额外配置 LLM 后端（详见下方"AI 配置"章节）

- **生成背调报告**：根据客户信息自动生成结构化调研报告（公司概况、关键发现、需求分析、合作价值）
- **智能问答**：针对特定客户提问，AI 结合已知信息给出建议
- **情报提取**：录入情报片段后，AI 自动提取关键信息

### 日历订阅

- 将 CRM 中的待办提醒导出为标准 iCalendar（.ics）文件
- 可导入 iPhone 日历或 Google Calendar
- 提供局域网订阅链接，手机扫码即可订阅

### Excel 导入 / 数据同步

- 上传 Excel 文件（.xlsx / .xls），自动导入客户信息
- 支持双向同步：CRM → Excel（同步导出当前数据）
- 智能匹配列名，兼容常见的表格标题格式

### 智能导入客户

- 输入公司名称或网址，系统自动推断国家、行业等信息
- 支持按域名后缀映射国家（如 `.de` → 德国、`.jp` → 日本）
- 支持按关键词映射行业

### iCloud 跨设备同步

- 启动时自动从 iCloud Drive 拉取最新数据库文件
- 运行过程中定时推送到 iCloud
- 适合在办公室电脑和笔记本之间共享数据
- 注意：这是基于文件复制的"伪同步"，同时只能有一台设备运行

### 回收站

- 删除客户时先移入回收站（软删除），可随时恢复
- 支持单个恢复 / 清空回收站
- 永久删除不可恢复

## AI 配置

系统支持三种 LLM 后端，可在 `config.py` 或通过环境变量配置。

### 方式一：LM Studio（推荐本地运行）

1. 下载安装 [LM Studio](https://lmstudio.ai/)
2. 加载模型并启动本地服务器（默认端口 1234）
3. 系统自动检测并连接

### 方式二：Ollama

1. 安装 [Ollama](https://ollama.com/)
2. 拉取模型：`ollama pull qwen2.5:7b`
3. 系统自动检测并连接

### 方式三：OpenAI 兼容 API

```bash
set OPENAI_API_KEY=sk-your-key
set OPENAI_BASE_URL=https://api.openai.com/v1
set OPENAI_MODEL=gpt-4o-mini
set LLM_BACKEND=openai
```

### 环境变量一览

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CRM_DB_PATH` | 数据库目录路径 | `data/` |
| `LM_STUDIO_URL` | LM Studio 地址 | `http://localhost:1234` |
| `OLLAMA_URL` | Ollama 地址 | `http://localhost:11434` |
| `OLLAMA_MODEL` | Ollama 使用的模型 | `qwen2.5:7b` |
| `OPENAI_API_KEY` | OpenAI API 密钥 | 空 |
| `OPENAI_BASE_URL` | OpenAI API 地址 | `https://api.openai.com/v1` |
| `OPENAI_MODEL` | OpenAI 使用的模型 | `gpt-4o-mini` |
| `LLM_BACKEND` | LLM 后端选择 | `auto`（自动检测） |

## 常见问题

**Q：数据库在哪里？**
A：默认在项目目录下的 `data/crm_reminders.db`。可通过环境变量 `CRM_DB_PATH` 自定义路径。

**Q：每次打开要重新上传表格吗？**
A：不需要。数据持久化存储在 SQLite 数据库中，关闭再打开数据依然在。

**Q：需要联网吗？**
A：基础功能完全不需要联网，所有数据都在本地。AI 分析功能如果使用 LM Studio 或 Ollama 也不需要联网；使用 OpenAI 则需要联网。

**Q：如何跨设备同步数据？**
A：系统支持 iCloud Drive 同步。在两台设备上分别配置好 iCloud，系统会在启动时自动拉取、运行中定时推送最新的数据库文件。

**Q：每个同事的数据会互相影响吗？**
A：不会。每个人的数据存在自己电脑上，完全独立。如需给同事使用，直接拷贝整个项目文件夹即可。

**Q：AI 分析用不了怎么办？**
A：确认已安装并启动了 LM Studio 或 Ollama，且模型已加载。在设置页面的 AI 配置部分可以测试连接状态。

**Q：Windows 通知不弹怎么办？**
A：确保系统开启了应用通知权限。定时任务只有在服务运行时才生效，关闭窗口后通知不会弹出。

**Q：如何给日历订阅？**
A：在系统的设置页面可以获取局域网订阅链接，复制到 iPhone 的"设置 → 日历 → 添加日历订阅"即可。
